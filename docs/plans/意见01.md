# Mobius Agent Phase 1 修正方案

## 核心判断

原Plan的骨架（monorepo结构、状态机、接口隔离、TDD）是合理的，但存在以下工程缺陷：

1. **P0 逻辑缺陷** — stagnant终止被标记为COMPLETED，预算跟踪是空实现
2. **P1 能力缺失** — EvalJudge和ToolExecutor全是mock，Loop跑的是假数字而非真代码
3. **P1 DSL不完整** — 缺少边界条件（古德哈特定律防护）和可验证的完成条件表达
4. **P2 过早设计** — Hermes Python安全层在Phase 1零代码，gRPC协议无调用方
5. **P2 Memory不完整** — 缺少知识过期机制，没有CLAUDE.md导出闭环

修正原则：**Phase 1 必须能真正写代码、跑测试、修Bug、记经验**。不是跑通一个打印日志的mock循环。

## 修正2: 预算跟踪实现 [P0]

### 问题


### 修正

将budget改为动态更新的有状态对象：

**文件**: `packages/core/src/loop/loop-state.ts`

```typescript
export interface LoopBudget {
  maxIterations: number;
  maxTokens: number;
  timeoutSeconds: number;
  iterationsUsed: number;    // ← 新增
  tokensUsed: number;        // ← 新增
  remaining(): { iterations: number; tokens: number; timeoutMs: number };
  consumeIteration(tokensThisIter: number): void;  // ← 新增
  isExhausted(): boolean;    // ← 新增
}

export function createLoopBudget(
  maxIterations: number,
  maxTokens: number,
  timeoutSeconds: number
): LoopBudget {
  const startTime = Date.now();
  return {
    maxIterations,
    maxTokens,
    timeoutSeconds,
    iterationsUsed: 0,
    tokensUsed: 0,
    consumeIteration(tokensThisIter: number) {
      this.iterationsUsed++;
      this.tokensUsed += tokensThisIter;
    },
    remaining() {
      const elapsedMs = Date.now() - startTime;
      return {
        iterations: Math.max(0, this.maxIterations - this.iterationsUsed),
        tokens: Math.max(0, this.maxTokens - this.tokensUsed),
        timeoutMs: Math.max(0, this.timeoutSeconds * 1000 - elapsedMs),
      };
    },
    isExhausted() {
      const r = this.remaining();
      return r.iterations <= 0 || r.tokens <= 0 || r.timeoutMs <= 0;
    },
  };
}
```

## 修正4: 实现基于真实命令执行的EvalJudge [P1]

### 问题

原Plan的EvalJudge是mock：`score = Math.min(0.3 + state.iteration * 0.3, 0.95)`。这不是EvalJudge，是FakeJudge。一个不能评估真实输出的Loop Engine只是数字游戏。

### 修正

新增 `CommandBasedJudge` 实现，基于DSL conditions执行真实命令：

**文件**: `packages/core/src/eval/command-judge.ts`（新增）

```typescript
import { exec } from "child_process";
import { promisify } from "util";
import { EvalJudge, EvalResult, LoopState } from "../loop/loop-engine.js";
import { LoopDefinition, EvalCondition, BoundaryCondition } from "../loop/loop-definition.js";
import { access } from "fs/promises";

const execAsync = promisify(exec);

interface ConditionResult {
  description: string;
  passed: boolean;
  suggestion?: string;
}

export class CommandBasedJudge implements EvalJudge {
  private definition: LoopDefinition;
  private cwd: string;
  private history: number[] = [];

  constructor(definition: LoopDefinition, cwd: string = process.cwd()) {
    this.definition = definition;
    this.cwd = cwd;
  }

  async evaluate(state: LoopState, taskOutput: string): Promise<EvalResult> {
    const evalConfig = this.definition.eval;
    const conditionResults: ConditionResult[] = [];

    // 1. 检查完成条件
    for (const condition of evalConfig.conditions) {
      const result = await this.checkCondition(condition);
      conditionResults.push(result);
    }

    // 2. 检查边界条件（古德哈特定律防护）
    const boundaryViolations: string[] = [];
    for (const boundary of evalConfig.boundaryConditions) {
      const ok = await this.checkBoundary(boundary);
      if (!ok) {
        boundaryViolations.push(boundary.description);
      }
    }

    // 3. 计算分数
    const passedCount = conditionResults.filter((r) => r.passed).length;
    const totalConditions = conditionResults.length;
    let score = totalConditions > 0 ? passedCount / totalConditions : 0.5;

    // 边界条件违反直接降分至0
    if (boundaryViolations.length > 0) {
      score = 0;
    }

    // 4. 检测stagnation
    this.history.push(score);
    const isStagnant = this.detectStagnation();

    const suggestions = conditionResults
      .filter((r) => !r.passed)
      .map((r) => r.suggestion || r.description);

    if (boundaryViolations.length > 0) {
      suggestions.unshift(`BOUNDARY VIOLATION: ${boundaryViolations.join("; ")}`);
    }

    return {
      score,
      reasoning: [
        ...conditionResults.map((r) => `${r.passed ? "PASS" : "FAIL"}: ${r.description}`),
        ...(boundaryViolations.length > 0
          ? [`VIOLATIONS: ${boundaryViolations.join("; ")}`]
          : []),
        `Score: ${score.toFixed(2)} (${passedCount}/${totalConditions})`,
      ].join("\n"),
      suggestions,
      isStagnant,
    };
  }

  private async checkCondition(condition: EvalCondition): Promise<ConditionResult> {
    try {
      switch (condition.type) {
        case "command_exit_code": {
          const { exitCode } = await this.exec(command.command);
          const passed = exitCode === condition.expected;
          return {
            description: condition.description,
            passed,
            suggestion: passed ? undefined : `Command exited ${exitCode}, expected ${condition.expected}`,
          };
        }
        case "command_output_contains": {
          const { stdout, exitCode } = await this.exec(condition.command);
          const passed = exitCode === 0 && stdout.includes(condition.expected);
          return {
            description: condition.description,
            passed,
            suggestion: passed ? undefined : `Output does not contain "${condition.expected}"`,
          };
        }
        case "file_exists": {
          const passed = await this.fileExists(condition.path);
          return {
            description: condition.description,
            passed,
            suggestion: passed ? undefined : `File ${condition.path} does not exist`,
          };
        }
        case "file_not_empty": {
          const exists = await this.fileExists(condition.path);
          // 简化：只检查存在性，完整版应检查size > 0
          return {
            description: condition.description,
            passed: exists,
            suggestion: exists ? undefined : `File ${condition.path} is empty or missing`,
          };
        }
        case "custom": {
          const { exitCode } = await this.exec(condition.script);
          return {
            description: condition.description,
            passed: exitCode === 0,
            suggestion: exitCode === 0 ? undefined : `Custom check failed with exit ${exitCode}`,
          };
        }
        default:
          return { description: condition.description, passed: false, suggestion: "Unknown condition type" };
      }
    } catch (err) {
      return {
        description: condition.description,
        passed: false,
        suggestion: `Check threw: ${(err as Error).message}`,
      };
    }
  }

  private async checkBoundary(boundary: BoundaryCondition): Promise<boolean> {
    try {
      switch (boundary.type) {
        case "file_count_min": {
          const { stdout } = await this.exec(`find ${boundary.path} -type f | wc -l`);
          const count = parseInt(stdout.trim(), 10);
          return count >= boundary.min;
        }
        case "no_file_deleted": {
          const { stdout } = await this.exec("git diff --name-status --diff-filter=D | wc -l");
          return parseInt(stdout.trim(), 10) === 0;
        }
        case "command_forbidden": {
          // 在taskOutput中检查是否有禁用命令
          // 实际应在ToolExecutor层面拦截，这里做二次检查
          return true; // 简化Phase 1实现
        }
        default:
          return true;
      }
    } catch {
      return false;
    }
  }

  private detectStagnation(): boolean {
    if (this.history.length < 3) return false;
    // 最近3轮分数变化 < 0.05 视为停滞
    const recent = this.history.slice(-3);
    const maxDiff = Math.max(...recent) - Math.min(...recent);
    return maxDiff < 0.05;
  }

  private async exec(command: string): Promise<{ stdout: string; stderr: string; exitCode: number }> {
    try {
      const { stdout, stderr } = await execAsync(command, { cwd: this.cwd, timeout: 30000 });
      return { stdout, stderr, exitCode: 0 };
    } catch (err: any) {
      return {
        stdout: err.stdout || "",
        stderr: err.stderr || "",
        exitCode: err.code || 1,
      };
    }
  }

  private async fileExists(path: string): Promise<boolean> {
    try {
      await access(path);
      return true;
    } catch {
      return false;
    }
  }
}
```

## 修正5: 实现真实的ToolExecutor [P1]

### 问题

原Plan的ToolExecutor是mock：

```typescript
execute: async (toolName, params) => ({
  success: true,
  output: `[Phase 1 mock] Tool ${toolName} executed...`,
  artifacts: [],
}),
```

一个不能写文件、运行命令的Agent不是Agent，是echo服务器。

### 修正

**文件**: `packages/core/src/executor/local-executor.ts`（新增）

```typescript
import { exec } from "child_process";
import { promisify } from "util";
import { readFile, writeFile, mkdir, access } from "fs/promises";
import { dirname } from "path";
import { ToolExecutor, ToolResult } from "../loop/loop-engine.js";

const execAsync = promisify(exec);

export interface LocalExecutorConfig {
  cwd: string;           // worktree根目录
  allowedCommands?: string[];  // 允许执行的命令白名单（可选安全）
  maxOutputLength?: number;    // 截断过长输出
}

/**
 * LocalExecutor — 在本地worktree中执行文件和命令操作。
 * Phase 1 的核心工具集：write_file, read_file, run_command。
 */
export class LocalExecutor implements ToolExecutor {
  private config: LocalExecutorConfig;

  constructor(config: LocalExecutorConfig) {
    this.config = {
      maxOutputLength: 50000,
      ...config,
    };
  }

  async execute(toolName: string, params: Record<string, unknown>): Promise<ToolResult> {
    try {
      switch (toolName) {
        case "write_file":
          return await this.writeFile(
            params.path as string,
            params.content as string
          );
        case "read_file":
          return await this.readFile(params.path as string);
        case "run_command":
          return await this.runCommand(
            params.command as string,
            (params.cwd as string) || this.config.cwd
          );
        case "append_file":
          return await this.appendFile(
            params.path as string,
            params.content as string
          );
        case "list_dir":
          return await this.listDir(params.path as string);
        default:
          return {
            success: false,
            output: `", // 不支持的命令
            error: `Unknown tool: ${toolName}. Available: write_file, read_file, run_command, append_file, list_dir`,
          };
      }
    } catch (err) {
      return {
        success: false,
        output: "",
        error: (err as Error).message,
      };
    }
  }

  private async writeFile(filePath: string, content: string): Promise<ToolResult> {
    const fullPath = this.resolvePath(filePath);
    await mkdir(dirname(fullPath), { recursive: true });
    await writeFile(fullPath, content, "utf-8");
    return {
      success: true,
      output: `Wrote ${filePath} (${content.length} bytes)`,
      artifacts: [{ type: "file", path: filePath, description: `Written ${filePath}` }],
    };
  }

  private async appendFile(filePath: string, content: string): Promise<ToolResult> {
    const fullPath = this.resolvePath(filePath);
    await mkdir(dirname(fullPath), { recursive: true });
    const fs = await import("fs/promises");
    await fs.appendFile(fullPath, content, "utf-8");
    return {
      success: true,
      output: `Appended to ${filePath} (${content.length} bytes)`,
    };
  }

  private async readFile(filePath: string): Promise<ToolResult> {
    const fullPath = this.resolvePath(filePath);
    const content = await readFile(fullPath, "utf-8");
    return {
      success: true,
      output: content,
    };
  }

  private async runCommand(command: string, cwd: string): Promise<ToolResult> {
    // 安全检查：禁止rm -rf /
    const normalized = command.trim().toLowerCase();
    if (normalized.includes("rm -rf /") || normalized.includes("rm -rf /*")) {
      return {
        success: false,
        output: "",
        error: "FORBIDDEN: Recursive delete of root is blocked",
      };
    }

    const { stdout, stderr, exitCode } = await this.execWithTimeout(command, cwd);
    const output = this.truncate(stderr ? `${stdout}\n---STDERR---\n${stderr}` : stdout);

    return {
      success: exitCode === 0,
      output,
      error: exitCode !== 0 ? `Exit code: ${exitCode}` : undefined,
    };
  }

  private async listDir(dirPath: string): Promise<ToolResult> {
    const fullPath = this.resolvePath(dirPath);
    const fs = await import("fs/promises");
    const entries = await fs.readdir(fullPath, { withFileTypes: true });
    const lines = entries.map((e) => `${e.isDirectory() ? "DIR " : "FILE"} ${e.name}`);
    return {
      success: true,
      output: lines.join("\n"),
    };
  }

  private resolvePath(filePath: string): string {
    if (filePath.startsWith("/")) return filePath;
    return `${this.config.cwd}/${filePath}`;
  }

  private async execWithTimeout(command: string, cwd: string, timeoutMs = 30000): Promise<{
    stdout: string;
    stderr: string;
    exitCode: number;
  }> {
    try {
      const { stdout, stderr } = await execAsync(command, { cwd, timeout: timeoutMs });
      return { stdout, stderr, exitCode: 0 };
    } catch (err: any) {
      return {
        stdout: err.stdout || "",
        stderr: err.stderr || "",
        exitCode: err.code || 1,
      };
    }
  }

  private truncate(str: string): string {
    const max = this.config.maxOutputLength || 50000;
    if (str.length <= max) return str;
    return str.slice(0, max) + `\n... [truncated ${str.length - max} chars]`;
  }
}
```

```typescript
 * Phase 1 简化版安全层。
 * 在 LocalExecutor 中通过命令白名单和关键字拦截实现。
 * Phase 2 迁移到独立Python进程（Hermes）。

export interface GuardrailRule {
  id: string;
  type: "tool_deny" | "param_validate" | "output_scan";
  pattern: RegExp | string;
  action: "block" | "warn" | "require_approval";
  description: string;
}

export class SimpleGuardrails {
  private rules: GuardrailRule[] = [];

  constructor() {
    // 默认规则
    this.addRule({
      id: "block-rm-rf-root",
      type: "tool_deny",
      pattern: /rm\s+-rf\s+\/\s*$/i,
      action: "block",
      description: "Block recursive delete of root filesystem",
    });
    this.addRule({
      id: "block-curl-pipe-sh",
      type: "tool_deny",
      pattern: /curl.*\|\s*sh/i,
      action: "block",
      description: "Block curl | sh execution",
    });
  }

  addRule(rule: GuardrailRule): void {
    this.rules.push(rule);
  }

  check(toolName: string, params: Record<string, unknown>): {
    allowed: boolean;
    violations: string[];
  } {
    const violations: string[] = [];
    const text = `${toolName} ${JSON.stringify(params)}`;

    for (const rule of this.rules) {
      const pattern = typeof rule.pattern === "string"
        ? new RegExp(rule.pattern, "i")
        : rule.pattern;

      if (pattern.test(text)) {
        if (rule.action === "block") {
          return { allowed: false, violations: [rule.description] };
        }
        violations.push(rule.description);
      }
    }

    return { allowed: true, violations };
  }
}
```

### 问题

原EventBus的`off()`是空实现，插件卸载时会导致内存泄漏。

```typescript
export class EventBus {
  private emitter = new EventEmitter();
  private handlerCount = 0;
  private handlerMap = new Map<string, { eventType: string; fn: (...args: any[]) => void }>();

  on(eventType: string, handler: EventHandler): string {
    const id = `handler-${++this.handlerCount}`;
    const wrapped = async (event: MobiusEvent) => {
      try {
        await handler(event);
      } catch (err) {
        console.error(`[EventBus] Handler ${id} error:`, err);
      }
    };
    this.emitter.on(eventType, wrapped);
    this.handlerMap.set(id, { eventType, fn: wrapped });
    return id;
  }

  once(eventType: string, handler: EventHandler): string {
    const id = `handler-${++this.handlerCount}`;
    const wrapped = async (event: MobiusEvent) => {
      try {
        await handler(event);
      } catch (err) {
        console.error(`[EventBus] Handler ${id} error:`, err);
      } finally {
        this.handlerMap.delete(id);
      }
    };
    this.emitter.once(eventType, wrapped);
    this.handlerMap.set(id, { eventType, fn: wrapped });
    return id;
  }

  off(handlerId: string): void {
    const registered = this.handlerMap.get(handlerId);
    if (registered) {
      this.emitter.removeListener(registered.eventType, registered.fn);
      this.handlerMap.delete(handlerId);
    }
  }

  // emit 保持不变
}
```

## Phase 1验收标准（修正后）

原验收标准：CLI运行mock循环，打印phase变化。

**修正后验收标准：**

1. 给定一个YAML Loop定义（如hello-world），包含真实的完成条件（如`npm test`通过）
2. 运行 `mobius run hello-world.loop.yaml -t "写一个hello world函数"`
3. Agent在worktree中实际创建文件（如`src/index.ts`、`src/__tests__/index.test.ts`）
4. 运行 `npm test` 验证代码
5. 如果测试失败，Loop继续迭代（修改代码 → 再次测试）
6. 直到所有conditions通过（score=1.0）或达到stagnant/budget上限
7. Memory中记录了每次迭代的facts、errors、patterns
8. 可以用 `mobius export-knowledge` 导出CLAUDE.md

这个验收标准意味着：**Phase 1结束时，你拥有的是一个能真正写代码、跑测试、修bug、记经验的Agent，而不是一个跑mock的数字框架。**
